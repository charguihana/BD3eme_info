BASE DE DONNÉES

Ensemble structuré de données organisées, stockées et manipulées efficacement.

1. BASE DE DONNÉES

Définition :

Une base de données est un ensemble structuré, organisé et persistant de données, relatives à un ou plusieurs domaines.

Caractéristiques :

  • Organisée (sous forme de tables)
  • Non redondante
  • Cohérente / Persistante
  • Relative à un domaine précis
2. TABLE

Définition :

Une table est un ensemble de données relatives à un même sujet. Elle est organisée sous forme de lignes et de colonnes.

Caractéristiques :

  • Chaque table décrit une entité
  • Composée d'attributs (colonnes)
  • et d'enregistrements (lignes)
3. ATTRIBUT (COLONNE)

Définition :

Un attribut est une propriété élémentaire décrivant une entité. Il correspond à une colonne dans une table.

Caractéristiques :

  • Possède un nom
  • Possède un type de données
  • Peut avoir des contraintes (NOT NULL, UNIQUE, DEFAULT, ...)
4. ENREGISTREMENT (LIGNE)

Définition :

Un enregistrement est une suite de valeurs qui correspond à une occurrence de l'entité décrite par la table.

Caractéristiques :

  • Une ligne dans une table
  • Contient une valeur pour chaque attribut
  • Représente un objet réel
5. CLÉ PRIMAIRE

Définition :

Attribut (ou ensemble d'attributs) qui permet d'identifier de manière unique chaque enregistrement d'une table.

Caractéristiques :

  • Unique (aucune valeur répétée)
  • Non NULL
  • Stable (ne change pas)
6. CLÉ ÉTRANGÈRE

Définition :

Attribut d'une table qui fait référence à la clé primaire d'une autre table. Elle assure la relation entre les tables.

Caractéristiques :

  • Assure l'intégrité référentielle
  • Permet de lier les tables
  • Évite la redondance des données
7. RELATION

Définition :

Lien logique établi entre deux tables afin d'éviter la redondance des données et d'assurer la cohérence.

Caractéristiques :

  • Basée sur les clés (primaire et étrangère)
  • Peut être de type 1-1, 1-N ou N-N
8. CONTRAINTE D'INTÉGRITÉ

Définition :

Règles qui garantissent la cohérence, la validité et la fiabilité des données dans la base.

Types principaux :

  • Intégrité de domaine (type et valeur valides)
  • Intégrité d'entité (clé primaire unique)
  • Intégrité référentielle (relations correctes)
